从左往右介绍
docker：仿真平台

fire_infer:视觉部分，启动launch文件，即可实现对靶标的识别和数字的辨别
           源码部分：
           infer_node.py：是视觉部分的总运行代码，负责识别靶标和辨别数字并且计算出数字
           的像素坐标，接着发布识别成功后的数字和其对应的像素坐标。

           listen_test.py：是视觉和控制的转接点，它监听来自infer_node.py节点推送的数
           字，像素坐标，和无人机的实时位置（监听mavros获取经纬度），将三者信息，同时储
           存在一个“框”里，一个数字对应一个像素坐标和一个经纬度，然后在处理，将“框”里的
           中位数首先提取出来，获取它的像素坐标和飞机那时的经纬度，根据公式，即可推算出
           目标数字的实际经纬度，最后发布出去。
          
           其余源码：上面两个主代码所用到的一些模块和功能，还有一些是测试用的。

px4_launch:仿真平台里集成的，存放launch文件，好复制到px4功能包里。

single_demo:控制部分，启动launch文件，即可实现控制无人机完成自主打击目标点（结合视觉部分）和返航任务
            源码部分（套了一个my_way功能包）：
            fixedwing.py:功能完善，自动拉取关键航点，组成完整的投水和降落航线（不稳定，
            很容易报错）

            fixedwing_waypoints.py：px4仿真用的，完整投水和降落航线需要自己手动敲进去
            （很稳定）。
            其余源码：模块和功能。

            zc_apm:这个就是可以上传到机载电脑上执行的源码，飞控固件是apm
            
整体说明：首先，我们需要在地面站上规划一条完整的起飞和降落航线，无人机会按照航线飞行，当
         无人机在靶标上空时，视觉部分就会先识别靶标，然后在识别数字，这里就在视觉部分
         了，识别成功后，为了给它充足的时间进行计算然后发布出置信度最高的数字经纬度坐
         标，我们让它在指定航点在发布，而控制部分节点一直在订阅这个目标经纬度，订阅成
         功，即可根据起飞点，目标点，降落点三个主要关键点组成新的航线，完成投水和降落。
